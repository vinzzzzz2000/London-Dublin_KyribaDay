CashAI V1 · Ceiling
03 / 07

But Prophet Has a Ceiling
What V1 Cannot Do

V1 was designed for automation, not intelligence. As treasury use cases grow more complex, the gaps become clear — and consequential.

⊘
No Sophisticated Forecasting
Cannot construct personalised, rule-based, or multi-driver forecasts beyond simple time-series extrapolation
⊘
No Explainability
Zero post-processing analysis — users cannot understand why a forecast was generated or what drove it
⊘
No Natural Language Interface
Users cannot express custom intent. Every parameter must be configured through UI — no conversational control
⊘
Single-Model Architecture
One algorithm for all cash flow types. No routing logic based on data characteristics, seasonality, or volatility profiles
⊘
▋
ESC or click outside to close
These gaps define the TAI mandate →

TAI · Trusted AI Architecture
04 / 07

The New Layer: TAI
From Prompt to Executable Code

How it works
  • Users express forecasting intent in plain natural language
  • TAI generates Python transformation code via agentic reasoning
  • Code is executed in an isolated, sandboxed environment
  • Every step is transparent and explainable — during and after the run
  • Same prompt always produces the same deterministic result
  • Average run time ~2 min; supports batch execution
Agent capabilities
SC CoT Reasoning Post Ranker Auto-Retry Cond. Caching Model Router
Transformation operations
set scale shift smooth seasonal_bump blend clamp condition aggregate residual_correction custom
Model routing logic
deterministicPure arithmetic / rule-based: fixed ratios, uplift, redistribution
prophet_regressorsWhen external drivers must be learned (holidays, weather, campaigns)
prophet_transformedPreprocessing required before forecasting (smoothing, capping, anomaly removal)
prophet_simpleDefault for plain extrapolation — no regressors or preprocessing
catboostMulti-feature tabular regression with categorical / numerical predictors
